Hybrid Rule Engine Adjustments (Post-ML)
After the ML model outputs a raw phishing probability, these deterministic rules adjust that score to reduce operational false positives and false negatives. Applied rules are logged for each scanned email.
−0.18
Trusted sender domain: If sender domain is in the known-legitimate list, phishing probability is reduced by 0.18 (clamped to ≥ 0.05).
+0.20
IP-based URL detected: URLs containing raw IPv4 addresses are strong phishing indicators; probability is increased by 0.20.
+0.12
Suspicious TLD: If sender domain ends with a suspicious TLD, probability is increased by 0.12.
+0.10
URL shortener detected: Presence of known shortener domains (bit.ly, tinyurl, etc.) increases probability by 0.10.
+0.06
Digits in sender domain: If an untrusted sender domain contains digits, probability is increased by 0.06.
